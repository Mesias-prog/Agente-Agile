Este proyecto contiene la documentación y planificación ágil para el desarrollo del MVP de gestión de turnos para un taller mecánico en Guayaquil.

## Estructura de carpetas
- `inbox/`: Evidencias de entrevistas y requerimientos (cliente, asesor, mecánico).
- `outputs/`: Artefactos generados (JSONs, Markdown, reportes HTML).
- `.claude/`: Configuración del agente y habilidades.

## Artefactos principales
- **Epics & Backlog**: Definición del alcance y priorización.
- **Sprint Plan**: Planificación del primer sprint.
- **Reporte**: Visor visual del estado del proyecto (`report.html`).
